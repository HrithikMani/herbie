Documentation

This section provides detailed documentation on how to use Herbie.

Getting Started

To get started with Herbie, follow these steps:

  1. Clone the project to your local machine or download the Chrome extension from the above link.
  2. Open Chrome and go to Preferences... -> Extensions.
  3. Ensure "Developer mode" is checked (upper left).
  4. Click "Load unpacked extension...".
  5. Browse to the `chrome_extension` folder in the project.

Features

Herbie offers several key features:

  • Behavior-Driven Development: Scripts are easy to understand and author.
  • Interactive Inspector: Discover elements on a page effortlessly.
  • mie-simulijs: Instead of using the jQuery Simulate Extended plug-in (a.k.a. jquery-simulate-ext) for simulating complex user interactions based on the jQuery.simulate() plug-in, we have used mie-simulijs, which was developed in MIE. It's a package that simulates events on the page. Here is the npm package: mie-simulijs.

Modules

Herbie is composed of several modules, each serving a specific purpose:

background.js

This module handles the background operations and interactions with the Chrome API.

commands.js

This module defines the commands that Herbie can execute, such as click, type, and navigate.

content.js

This module interacts with the content of the web pages and handles script injections and event listeners.

content-utils.js

Utility functions used by content scripts to manipulate the DOM and handle events.

Dragdrop.js

This module simulates drag and drop events on web pages.

ExecuteScript.js

This module executes the generated scripts and simulates user interactions on the page.

FindElement.js

This module includes functions to locate and interact with DOM elements based on various criteria.

keywords.js

This module defines the keywords used in the Behavior-Driven Development scripts.

logs.js

This module handles logging of actions and results for debugging and review purposes.

main.js

The main entry point for the Herbie extension, initializing and managing the overall workflow.

parser.js

This module parses the BDD scripts and translates them into executable commands.

record.js

This module records user interactions on the page to generate BDD scripts.

scripts.js

This module manages the execution of scripts within the Herbie extension.

utils.js

Utility functions used across different modules to perform common tasks.

Examples

Here are some examples of how to use Herbie:

Example 1: Click an Element


        // BDD Script to click an element
        Given the user is on the homepage
        When the user clicks the "Login" button
        Then the login form should be displayed

Example 2: Fill a Form


        // BDD Script to fill out a form
        Given the user is on the signup page
        When the user enters "John Doe" into the "Name" field
        And the user enters "john.doe@example.com" into the "Email" field
        And the user clicks the "Submit" button
        Then the signup confirmation message should be displayed
