Documentation

This section provides detailed documentation on how to use Herbie.

Getting Started

To get started with Herbie, follow these steps:

  1. Clone the project to your local machine or download the Chrome extension from the above link.
  2. Open Chrome and go to Preferences... -> Extensions.
  3. Ensure "Developer mode" is checked (upper left).
  4. Click "Load unpacked extension...".
  5. Browse to the chrome_extension folder in the project.

Herbie

Herbie is a powerful Chrome extension designed to automate and test web interactions seamlessly. It leverages Behavior-Driven Development (BDD) principles, making it accessible for both developers and non-developers to write and understand test scripts.

Click

The click command is used to simulate a mouse click on a specified element. You can define an XPath, class, or ID for the element you want to interact with, and use this keyword in your scripts. For example:

click on "//*[@id='submit-button']"

This command will click on the element with the ID 'submit-button'. Alternatively, you can pass a class:

click on "//*[@class='button-class']"

This command will click on the first element with the class 'button-class'.

Type

The type command is used to simulate typing text into an input field. You specify the text to be typed and the target element's XPath, class, or ID. For example:

type "Hello, World!" in "//*[@id='text-input']"

This command will type "Hello, World!" into the input field with the ID 'text-input'. You can also target an element by class:

type "Hello, World!" in "//*[@class='input-class']"

This command will type "Hello, World!" into the first input field with the class 'input-class'.

Wait

The wait command pauses the script execution for a specified amount of time, which is useful for handling asynchronous operations on the webpage. For example:

wait 5000

This command will pause the script for 5000 milliseconds (5 seconds) before proceeding to the next command.

Verify

The verify command is used to check if a particular text is present in a specified element on the page. You can use XPath, class, or ID selectors to define the target element. If the text is not found, the script will stop execution. For example:

verify "Success" in "//*[@id='message']"

This command will verify that the text "Success" is present in the element with the ID 'message'. You can also target an element by class:

verify "Success" in "//*[@class='message-class']"

This command will verify that the text "Success" is present in the first element with the class 'message-class'. If the text is not found, an error is logged and the script stops.

You can manage these keywords and more within the "Keywords" tab of the Herbie extension interface. This flexibility allows you to target elements using a variety of selectors, making your scripts more adaptable to different web page structures.
